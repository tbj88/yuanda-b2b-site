# Yuanda 外贸 B2B 品牌官网完整本地展示版交接说明

## 1. 本次交付范围

本次交付为**完整本地展示版**，未部署 GitHub Pages。站点根据 Alibaba.com 店铺链接整理生成，视觉方向为“工业高端”，适用于商超货架、仓储架、购物车等 B2B 工程采购场景。

来源链接：<https://ydhj.en.alibaba.com/>

## 2. 文件结构

| 文件/目录 | 用途 |
|---|---|
| `index.html` | 单文件 SPA 页面，包含样式、交互、多语言、staff/admin 模式 |
| `data.json` | 网站全部可编辑内容，包括品牌、产品、资质、展会、区域、内部 SOP |
| `hero/` | 后续自托管 Hero 大图目录 |
| `logos/` | 后续自托管客户 Logo 目录 |
| `textures/` | 材质或背景纹理目录 |
| `uploads/` | 管理后台上传图片目录，部署阶段使用 |
| `ai-previews/` | 可选 AI 实景图目录 |
| `交接说明.md` | 当前说明文档 |

## 3. 如何本地预览

在 `yuanda-b2b-site` 目录启动静态服务：

```bash
python3 -m http.server 8080
```

然后打开：

- 客户公开页：`http://localhost:8080/`
- 业务员视图：`http://localhost:8080/?staff`
- 管理员视图：`http://localhost:8080/?admin`

业务员视图本地软密码建议使用：`yuanda2026`。当前本地版本只要输入任意非空字符即可解锁，正式部署时可改成固定密码逻辑。

首次进入管理员视图会要求输入 token；本地预览阶段任意输入一段字符即可，它会写入浏览器 `localStorage.yuanda_admin_token`。

## 6. 后续完整部署需要

1. GitHub 账号
2. 公开仓库，例如 `yd` 或 `yuanda`
3. GitHub Pages 开启 main/root
4. fine-grained PAT，仅授权该仓库 Contents 读写
5. 将阿里图片下载或替换为自托管图片，避免外链失效

## 7. 视图权限说明

| URL | 对象 | 权限 |
|---|---|---|
| `/` | 客户/买家 | 只读公开官网 |
| `/?staff` | 内部业务员 | 公开内容 + 情报画像 + 话术 SOP，只读 |
| `/?admin` | 老板/主管 | 需要 token，显示本地原型编辑器 |

- 当前为完整本地展示版，不等同于已上线生产站。
- 正式上线前，不建议继续引用 Alibaba 图片外链，应迁移到本地仓库目录。
- 页脚没有放 Alibaba 店铺链接，符合品牌官网定位。
- 公开页不会显示编辑按钮或内部 SOP。
